# 编码实践 · 总览

面向 **开发者与评审**：说明本案例在 **AI 辅助编码** 下的工程分层、对话模拟实现要点、人机分工与常见避坑。详细内容已 **拆分** 为下列子文档，可按角色选读。

| 顺序 | 文档 | 适合谁 | 内容提要 |

**领导快速了解（1 分钟）**：本案例在 **预设三栏工作台 + 业务组件库** 上，用 AI 完成了 **流式对话模拟、三处阻塞、多步思考与报告收尾**；**模拟逻辑与 UI 分层**，后续换真实接口主要改数据层。人类修正 **主要集中在流程与状态时机**，设计类返工在 **有稿 + MCP + 规则** 时相对较少。细节见 **03** 与 **04**。

## 本阶段中 AI 与人类的粗分工（摘要）

| 侧重 | AI | 人类 |
|------|-----|------|
| 产出 | 页面组合、模拟类型与配置、流式工具、Hook 草稿、按指令修 Bug | 拍板剧本顺序与阻塞类型、验收主路径、防止误改通用组件库 |
| 工具 | 宿主项目中的 Rules、生成页面类 Skill、组件库 MCP、Figma 类 MCP | 配置 MCP、维护规则、关键交互的文字时序说明 |

更完整的工具与价值判断见 [《03-AI协作与工程化底座》](./03-AI协作与工程化底座.md)。
