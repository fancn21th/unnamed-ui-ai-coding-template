# AI Coding 人员实践产物清单

本套资料按 **需求 → 设计 → 编码 → 测试 → 产品化/上线** 分目录存放，可单独打包上传。各阶段文档内附有 **「AI 在本阶段的作用」** 与 **Rules / Skills / MCP** 的说明（均以通用名指代，不绑定某一仓库路径）。

| 阶段 | 文件夹 | 主要内容 |
|------|--------|----------|
| 需求 | `requirements/` | 需求文档（PRD）、需求实践检查项 |
| 设计 | `design/` | 设计说明、界面与组件映射 |
| 编码 | `coding/` | 《编码实践》总览 + `01`–`04` 专题（架构、模拟与状态、AI 底座、避坑与演进）、`examples/` |
| 测试 | `testing/` | 测试用例文档、Vitest 用例与 `fixtures/` |
| 产品化/上线 | `release/` | 构建、检查与发布前清单 |

## AI 工具索引（概念说明，无仓库路径）

在 **Cursor**（或同类 IDE）中，由产品加载 **Rules**、按需触发 **Skills**、连接 **MCP**；人类负责范围、验收与合并。下表仅说明 **用途**，具体文件名与配置以你所在项目为准。

| 类型 | 常见命名 / 位置习惯 | 典型用途 |
|------|---------------------|----------|
| **Rules** | 如「项目概述」规则 | 约定技术栈、组件优先级、源码别名，减少生成代码跑偏 |
| **Rules** | 如「组件修改约束」 | 优先在业务层改样式，少改设计系统/底层组件库源码 |
| **Rules** | 如「设计 Token」 | 用语义化颜色与间距，避免魔法值 |
| **Rules** | 如「组件使用顺序」 | composed → blocks → ui 等选用顺序 |
| **Rules** | 如「页面生成质量」 | 结构、可访问性、命名等生成要求 |
| **Skill** | 如「生成页面」类技能 | 拆需求 → 蓝图 → 组件选型 → 实现顺序 |
| **Skill 参考** | 组件清单、示例页说明 | 与 MCP 互补的静态选型材料 |
| **MCP** | 组件库服务 | 查询组件 API、props、目录结构 |
| **MCP** | Figma 桌面类服务 | 拉取设计稿节点、截图与上下文；注意选区大小与 token 限制 |

## 脱库使用说明

- **Markdown**：已避免指向宿主仓库内的 `.cursor`、`docs`、`src` 等路径；仅保留本套目录之间的相对关系（如 `requirements/` 与 `design/`）。
- **测试代码**：`testing/*.test.ts` 仅引用本目录内 `fixtures/`，上传后若在 **无宿主工程** 的环境下跑测试，需自备 `vitest` 与最小 `package.json`；若在 **原工程** 内跑，由宿主 Vite/Vitest 配置包含本目录即可。
- **示例代码**：`coding/examples/*.ts` 不使用路径别名，可直接复制进任意工程再改类型归属。
