# 产品化与上线检查

本套资料对应的案例定位为 **实验 / 演示**，以下清单可按实际上线目标裁剪；正式对外发布建议全量执行并签字。

---

## 0. 本阶段中 AI 的作用与工具（产品化 / 上线）

| 角色 | 说明 |
|------|------|
| **AI 适合做什么** | **起草** 本清单各节、项目 README 中的脚本说明、`.env.example` 模板；根据 `package.json` **生成** CI 工作流草稿（如安装依赖 + 构建 + 测试）；整理「演示 vs 生产」对外话术。 |
| **人类必须做什么** | 选定环境与门禁（是否要求全量 typecheck/lint）；管理密钥与流水线凭据；发布与回滚决策。 |

| 工具 | 在本阶段如何用上 |
|------|------------------|
| **Rules** | 让 AI 更新文档时遵守「项目概述」类规则，避免 README 与真实脚本（例如仅 Vite 构建、typecheck 单独说明）矛盾。 |
| **Skills** | 若上线含新页面，可将「生成页面」类技能中的自检项并入发布前手工列表。 |
| **MCP** | 非必须；若交付需附组件来源说明，可用组件库 MCP 辅助摘录（按需）。 |

**说明**：本 `list/` 包内文档可由 AI 起草、人类定稿；发布闸门以本文第 1 节为准。

---

## 1. 构建与质量闸门

| 检查项 | 命令 / 说明（示例） | 备注 |
|--------|---------------------|------|
| 依赖安装 | `pnpm install`（或 npm/yarn，以宿主为准） | CI 与本地一致 |
| 生产构建 | `pnpm build` | 失败不得发布 |
| 单元测试 | `pnpm test`（若已接入 Vitest） | 可包含本套 `list/testing` 用例 |
| 类型检查 | `pnpm typecheck` | 是否与构建分离，以宿主说明为准 |
| 代码风格 | `pnpm lint` | 是否阻断发布由团队约定 |

---

## 2. 运行时与配置

| 检查项 | 说明 |
|--------|------|
| 环境变量 | 敏感信息不入库；用 `.env.example` 列出变量名 |
| 静态资源 | 外链是否长期可用、是否需自有 CDN |
| 路由与 Base | 子路径部署时配置构建工具的 `base` |

---

## 3. 安全与合规（按需）

- 依赖漏洞扫描（`pnpm audit` 或流水线）。
- 第三方图片域名与 CSP、隐私说明。
- 若收集用户数据：合规与日志脱敏。

---

## 4. 可观测与运维（按需）

- 前端错误上报是否在正式环境开启。
- 版本号与变更记录或 Tag 一致。

---

## 5. 文档与交付

| 检查项 | 说明 |
|--------|------|
| 项目 README | 启动方式、脚本、已知限制 |
| 本套 `list/` 产物 | 各阶段文档齐全、可单独交付 |
| 演示说明 | 标明 Mock，避免误解已接生产 API |
