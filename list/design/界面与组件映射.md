# 界面与组件映射

将界面区块映射到 **业务组件库**（示例体系：**composed** → **blocks** → **ui**）。下表组件名为示例，宿主工程以实际包为准。

## 0. AI 如何维护本表

| 做法 | 说明 |
|------|------|
| **推荐** | 在 IDE 中通过 **组件库 MCP** 查询真实组件名、导出路径与 props，再更新下表；与「组件使用顺序」类 Rule 一致。 |
| **备选** | 无 MCP 时，对照团队内部的组件清单文档与源码目录手工更新。 |
| **AI 产出** | 适合 **首版草稿** 与改稿后批量重扫；合并前人类核对 import 与重命名。 |

**交叉校验**：本表可与 Figma 布局对照是否漏映射；编码阶段应遵守「少改底层组件库」类约定，单页需求优先业务层组合与 `className`。

---

## 1. 全局布局

| 界面区块 | 组件（示例） | 说明 |
|----------|--------------|------|
| 顶栏组合 | `PageHeader`、`PageHeaderButtonGroup`、`PageHeaderUser`、`PageHeaderButton` | composed + 对应 blocks |
| 三栏分割 | `TripleSplitPane` | 左/中/右可折叠 |

---

## 2. 分栏页面（示例职责）

| 栏 | 模块职责 |
|----|----------|
| 左 | 数据来源面板（列表、上传区等） |
| 中 | 对话面板（消息、输入、流式与阻塞 UI） |
| 右 | 工作空间面板（报告、任务卡片等） |

（具体文件名由宿主工程决定，本表不绑定路径。）

---

## 3. 对话相关（典型能力 → 组件示例）

| UI 能力 | 可能使用的 composed（以实际为准） |
|---------|-----------------------------------|
| 消息列表 | `MessageList`、`Message` |
| 输入发送 | `Sender` / `ResponsiveSender` |
| Markdown | Markdown 组合件 |
| 思考步骤 | `ThinkingProcess`、`ThinkingStepItem` |
| 任务/目标卡片 | `TaskCard`、`GoalCard` |
| 报告展示 | `ReportCard` |
| 建议入口 | `Suggestion` |

新增界面时，先用 **组件库 MCP** 或内部组件文档查 API，再落笔。

---

## 4. 模拟逻辑（非 UI，但影响表现）

| 能力 | 说明 |
|------|------|
| 阶段配置 | 剧本数组（如流式文案、延时、阻塞类型） |
| 类型定义 | 阶段的联合类型（text / delay / block / instant 等） |
| 流式/延时 | 纯函数：按 tick 输出文本、Promise 延时 |
| 状态机 | Hook：驱动阶段推进与用户确认 |

---

## 5. 映射维护约定

- 产品改稿后，同步更新「界面区块 → 组件」行。
- 替换组件时在变更说明中写明对应章节。
