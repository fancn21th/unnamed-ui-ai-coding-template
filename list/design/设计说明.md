## 1. 设计目标

1. **信息架构清晰**：三栏职责固定，用户形成稳定心智模型（数据从哪来 → 对话发生什么 → 产出在哪看）。
2. **对话为主视觉焦点**：中间栏占据主要注意力，流式与步骤反馈可读、不拥挤。
3. **与设计系统一致**：颜色、间距、圆角等优先使用语义化 Token；对齐设计稿时可借助 Figma 类工具拉取节点上下文。

---

## 2. 信息架构（IA）

```
┌─────────────────────────────────────────────────────────────┐
│  PageHeader（Logo + 标题「智能助手」+ 协作 + 用户）          │
├──────────┬──────────────────────────────┬──────────────────┤
│ 数据来源  │           对话                  │    工作空间       │
│          │  · 空状态 / 欢迎 / 建议          │  · 报告/任务卡片  │
│  列表等   │  · 消息列表（用户 / 助手）        │  · 占位内容       │
│          │  · 输入区（Sender）              │                  │
└──────────┴──────────────────────────────┴──────────────────┘
```

- **左栏**：宽约 240px（可折叠）；窄屏下可用 Popover 等承载同源内容。
- **中栏**：主区域伸缩，保证最小可读宽度（示例约 280px）。
- **右栏**：宽约 360px（可折叠为窄条）。

实现上：在应用根布局中配置三栏分割组件（如 `TripleSplitPane` 同类件）的左/中/右子面板。

---

## 3. 对话区关键界面状态

| 状态 | 说明 | 主要组件/区域（示例名） |
|------|------|-------------------------|
| 空状态 | 无消息时的引导与建议 | 欢迎区、`Suggestion` 等 |
| 流式回复 | 助手消息逐字/逐段出现 | `MessageList`、Markdown 渲染 |
| 阻塞-待办 | 用户需确认任务列表 | `blockType: task_list` |
| 阻塞-确认 | 模板/流程确认 | `blockType: confirm_panel` |
| 阻塞-补充 | 表单补充信息 | `blockType: supplement_form` |
| 思考步骤 | 多步执行过程 | `ThinkingProcess` 等 |
| 报告收尾 | 报告卡片 + 说明文案 | 报告卡片 + 文本流 |

状态由 **对话模拟配置**（阶段数组）驱动，与 UI 通过 Hook 衔接。

---

## 4. 视觉与 Token 原则

- 页面背景：使用设计系统语义变量（示例：`--Container-bg-neutral-light` 一类）。
- 间距：优先 Gap / Padding 类语义变量。
- 顶栏按钮：使用既有 Button 封装，业务层用 `className` 做圆角、高度微调；避免为单页改底层组件默认值（与「组件修改约束」类 Rule 一致）。
